三阶段实验计划

1
YOLO 人体识别 + 程序控制跟随 进行中 · 主线

车上实时检测人 → 规则算转向/速度 → 自动跟随。先纯感知(不控车),再跨安全线发 CAN 控车。这是整个项目的地基,也是 Phase 2 的数据来源和性能基线。

关键任务

  • YOLO 车载部署 — yolo11n → ONNX → TensorRT FP16,在 Xavier 求 15–30 fps
  • 实时感知链路 — 订阅相机 ROS 话题,逐帧检测人、选最大框为目标
  • 规则控制律 — 转向 steer = Kp·(cx/W − 0.5);速度 = 固定巡航 + 框高 P 控制;丢框/过近 → 停
  • CAN 控车(跨安全线)— ctrl_cmd_98c4d2d0,转向取负,SocketCAN 发 can0,安全护栏 + 人工点头

产出

  • 车上实时自动跟随 demo + 一套可复用的规则基线

数据采集要求(重新采集,补当前数据集短板)

  • 多样性 — 多路线 / 多光照 / 多人物 / 多衣着,目标 30–60 min,分多段 3–5 min/run
  • 制造速度信号 — 司机刻意随距离变速(人远了加速追、近了减速),否则速度永远是常数、学不出
  • 制造干净停车 — 让人主动停下 / 走到障碍后,司机随之停,形成"画面有线索 → 停"的因果对(为 v2 的 go/stop 头备料)
  • 裁掉首尾 — 每个 run 去掉起步前、收尾后的 artifact 段,避免污染标签
  • 双目视差(可选)— 给目标真实距离(米),替代不可靠的框高,让速度/停车有物理依据

训练设置

  • 验证集 — 按 run / 按时间段切分,严禁随机切(相邻帧泄漏会让指标虚高)
  • 预处理 — 1280×720 按比例 crop 后 resize,避免压扁;按固定时间间隔重采样 5 帧序列
  • 正则 — 数据量小,缩小通道 / dropout / 数据增强(水平翻转须同步翻转转向符号)
  • 评估 — 转向预测 vs 真值曲线 + 接入现有 follow_sim 做闭环回放;stop 看 recall 而非准确率

3
VLA 语义跟随 探索

从"跟最大框"升级到可用语言指令驱动的语义跟随:"跟着穿红衣服的人""到路口停下""慢一点跟"。Vision-Language-Action 把感知–推理–控制端到端打通。

方向

  • 候选模型 — OpenVLA、π0 (Physical Intelligence)、RT-2 类,或 TinyVLA / 小型可蒸馏 VLA
  • 价值 — 自然语言指令、零样本泛化到新目标、理解场景语义,超越几何跟随
  • 落地路径 — 先云端/离线验证可行性,再考虑量化 + 蒸馏塞进 Xavier,或边缘–云协同

主要挑战

  • 算力 — Xavier (~32 TOPS) 跑不动 7B 级 VLA 实时,必须裁剪/量化或下放算力
  • 数据 — 需要带语言标注的 (指令, 图像, 动作) 三元组,采集与标注成本高
